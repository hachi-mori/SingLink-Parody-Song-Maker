# 替え歌ロジック設計（vvproj 駆動化・実装前確定版）

最終更新: 2026-03-07

## 1. この設計の目的

- 問題文、問題数、問題ごとの最大入力音節数をプログラムから分離し、`Score/*.vvproj` から取得する。
- 既存シーン遷移と合成フローを維持したまま、替え歌ロジックを vvproj 駆動化する。
- この文書だけ読めば、実装者が仕様解釈で迷わず実装できる状態にする。

注記:
- 本ドキュメントは設計のみ。まだコード変更は行わない。

## 2. 用語定義

- 範囲指定テキスト: `talk` 前半に置く、置換対象を示すテキスト。例: `さかな[が]`
- 問題文テキスト: `talk` 後半に置く、画面表示用テキスト。例: `やすいおさかなは？`
- ベース語: 範囲指定テキストから `[` と `]` を除いた文字列（`[]` 内文字は残す）。例: `さかな[が]` -> `さかなが`
- 助詞: `[]` 内に書かれた文字列。例: `さかな[が]` -> `が`
- 最大入力音節数: ベース語を `splitSyllables` した音節数

## 3. vvproj データ契約

### 3.1 talk 配列の構成

- `ExtractTalkUtterances(vvprojPath)` が返す配列を `talkLines` とする。
- `talkLines.size()` は偶数であることを前提とする。
- 問題数 `N = talkLines.size() / 2`。
- `i` 問目（0-based）のデータ:
  - 範囲指定テキスト: `talkLines[i]`
  - 問題文テキスト: `talkLines[i + N]`

### 3.2 例

`talkLines = [R1, R2, R3, Q1, Q2, Q3]` のとき:
- 問題1: 範囲 `R1`, 表示文 `Q1`
- 問題2: 範囲 `R2`, 表示文 `Q2`
- 問題3: 範囲 `R3`, 表示文 `Q3`

### 3.3 品質前提

- vvproj は整形式で渡される前提。
- `talk` の偶数件、`[]` 記法の妥当性はデータ作成側で担保。
- 将来的に問題作成用アプリで担保する想定。

## 4. 画面ロジック仕様（WriteLyrics 側）

### 4.1 問題表示

- `m_topics` のハードコード表示は廃止。
- 各問題で表示する文字列は `talkLines[i + N]`（問題文テキスト）。
- 問題文テキストには番号を含めない。
- 画面表示時にプログラム側で `① `, `② `, ... を先頭付与する。
  - 例: `やすいおさかなは？` -> 表示 `① やすいおさかなは？`

### 4.2 入力制約（現行維持）

- ひらがな + `ー` のみ許可。
- 先頭 `ー` 禁止。
- `っ` 禁止。
- 長音 `ー` は現行ロジックで母音化（`replaceChoonWithVowel`）。
- 最小入力音節数は固定 `2`。
- 最大入力音節数は問題ごとに動的。

### 4.3 最大入力音節数の計算

- 範囲指定テキストを解析してベース語を得る。
- ベース語に対して `splitSyllables` を実行。
- その件数を `maxSyllable` とする。

### 4.4 問題文フォントサイズ（自動決定）

- 既存実装では各問題を人力指定していたが、新設計では文字列長から自動決定する。
- 判定対象は「番号付与前の問題文テキスト長」とする。
- 初期ルール（既存サイズ感に合わせる）:
  - 文字数 `<= 8` -> `fontSize = 100`
  - 文字数 `9〜10` -> `fontSize = 90`
  - 文字数 `>= 11` -> `fontSize = 80`
- これは既存5問の見た目に寄せるための初期値。新規問題で崩れる場合は閾値を再調整する。

## 5. `[]` 記法仕様

### 5.1 基本

- 記法は範囲指定テキストでのみ利用する。
- `[]` がある場合のみ助詞追加候補が存在する。
- `[]` がない場合、助詞追加候補は空。

### 5.2 解析結果

範囲指定テキストから次を生成する。
- `baseText`: `[` と `]` を除いた文字列（`[]` 内文字は残す）
- `particleText`: `[]` 内文字列（なければ空）

例:
- `さかな[が]` -> `baseText=さかなが`, `particleText=が`
- `さかな` -> `baseText=さかな`, `particleText=`(空)

### 5.3 利用箇所

- 最大入力音節数計算: `baseText` を使う。
- song 置換ターゲット比較: `baseText` を使う。
- 補完時の助詞追加: `particleText` を使う。

## 6. 補完アルゴリズム仕様

### 6.1 前提

- `inputSyllableCount` は正規化後入力の音節数。
- 受理条件は `2 <= inputSyllableCount <= maxSyllable`。

### 6.2 生成手順

1. `result = normalizedInput` で開始。
2. `particleSyllableCount = splitSyllables(particleText).size()`。
3. `remaining = maxSyllable - inputSyllableCount`。
4. 助詞があり、かつ `particleSyllableCount <= remaining` の場合:
- `reserveForParticle = particleSyllableCount`。
- `needVowelFill = remaining - reserveForParticle`。
- 末尾音節母音補完を `needVowelFill` 回追加。
- 最後に `particleText` を追加。
5. 助詞がない場合:
- 末尾音節母音補完を `remaining` 回追加。
6. 助詞があるが `particleSyllableCount > remaining` の場合:
- 助詞追加はスキップ。
- 末尾音節母音補完を `remaining` 回追加。

### 6.3 例

- 助詞=`が`, 入力=`たい`, 最大=4 -> `たいいが`
- 助詞=`より`, 入力=`たい`, 最大=4 -> `たいより`
- 助詞なし, 入力=`たい`, 最大=4 -> `たいいい`

### 6.4 タイムアップ

- その問題の `maxSyllable` 個ぶん `ら` を生成して回答とする。

### 6.5 表示用入力文字列の扱い

- `SolvedTask.userInput` は「ユーザーが入力した文字列（補完前）」を保持する。
- 置換で使う最終音節列は `SolvedTask.userSyllables` に保持する。
- これにより、Result 画面の入力表示はユーザー入力基準を維持しつつ、song 置換は補完後音節列で正しく実行できる。

## 7. 置換アルゴリズム仕様（ApplyParodyLyrics 側）

### 7.1 仕様

- 置換対象は `song.tracks[*].notes[*].lyric` の非空列。
- 問題 `i` のターゲットは範囲指定テキストの `baseText` 音節列。
- 置換文字列はユーザー回答の音節列。

### 7.2 重複ターゲット対応（必須）

- 同一ターゲットが複数問題で出る場合は「問題順 = 曲中出現順」で対応づける。
- 実装では「何回目の出現を処理済みか」を保持する。

推奨管理方法:
- キー: ターゲット音節列（または正規化文字列）
- 値: 既に消費した出現回数
- 置換時は `k` 回目の出現を探して置換し、消費回数を `k+1` に更新

### 7.3 現行との差分

- 現行 `OverwriteSequenceOnce` は常に先頭一致1件を置換するため、
  置換後も同じ語形だと同位置再置換の可能性がある。
- 新仕様では出現回数管理でこの曖昧さを除去する。

## 8. 責務分担方針（Scene と VOICEVOX）

### 8.1 判断基準

- `VOICEVOX` に置く処理:
- vvproj 構造の解釈、`talk`/`song` の抽出・変換、置換アルゴリズム
- 複数シーンで再利用可能なテキスト解析（`[]` 解析など）

- `Scene` に置く処理:
- 画面状態、入力受付、タイマー、遷移、表示メッセージ
- そのシーンでしか使わない UI 固有の分岐

### 8.2 この案件での推奨

- `VOICEVOX` に追加してよい処理（推奨）:
- 範囲指定テキスト解析（`baseText` / `particleText`）
- `talk` から問題セットを組み立てる補助
- 出現回数管理付き song 置換

- `WriteLyrics` に残す処理:
- 入力制約チェック
- 補完実行の呼び出しと結果保存
- 問題進行、タイムアップ、エラー表示

理由:
- vvproj 解釈を Scene に散らすと肥大化しやすい。
- 解析系を `VOICEVOX` に寄せるとテストしやすく責務分離しやすい。

## 9. 実装前チェックリスト（承認後実装用）

- `talkLines.size()` が偶数前提で問題数計算できること。
- 問題 `i` と `i+N` の対応で画面表示と置換対象が一致すること。
- `[]` 除去後文字列で最大音節数が算出されること。
- 助詞スキップ条件（助詞音節数 > 残り枠）が機能すること。
- タイムアップ `ら` 件数が問題ごとに変わること。
- 重複ターゲットが問題順で曲中出現順に置換されること。
- 現行入力制約（ひらがな、先頭 `ー`、`っ`、長音処理）が保持されること。
- 韻スコア関連データを削除しないこと。

## 10. 軽量実装順序（承認後）

1. `VOICEVOX` に `[]` 解析ユーティリティを追加。
2. `WriteLyrics` の問題データ取得を `talk` 前半/後半モデルへ変更。
3. 問題表示時に `① `, `② `, ... を付ける処理を追加。
4. 文字数ベースの `fontSize` 決定処理を追加。
5. `WriteLyrics` の最大音節数計算と補完処理を新仕様に置換。
6. `ApplyParodyLyrics` を出現回数管理付きに変更。
7. タイムアップ処理を `maxSyllable` 連動に変更。
8. 表示と置換結果の整合を確認。
