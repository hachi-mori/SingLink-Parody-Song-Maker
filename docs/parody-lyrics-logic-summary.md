# 替え歌周辺処理まとめ（現行仕様）

最終更新: 2026-03-07

## 1. この文書について

この文書は、現在の実装コードに一致する「替え歌周辺の処理」を後から参照しやすい形でまとめたものです。  
人間とAIが同じ前提で読めるように、処理の流れとデータ契約を中心に記述しています。

- 既存挙動の確認
- 新曲追加時のデータ作成ルール確認
- 改修時の影響範囲把握

---

## 2. 全体フロー

1. `talk` から問題データ（置換対象と問題文）を作る。
2. `WriteLyrics` で問題ごとに入力を受け付ける。
3. 入力文字列を制約チェックし、必要なら音節補完する。
4. `SolvedTask` に「表示用入力」と「置換用音節列」を保存する。
5. `ApplyParodyLyrics` で `song.notes.lyric` を置換し、歌唱用 vvproj を作る。
6. `BuildResultDisplayLyrics` で Result 表示用歌詞を作る。

---

## 3. vvproj の参照ルール

### 3.1 `talk` の扱い

- `ExtractTalkUtterances(vvprojPath)` で `talk` の `audioKeys` 順に文字列を取得する。
- 取得配列を `talkLines` とする。
- `talkLines.size()` は偶数を前提とする。
- 問題数は次で決まる。  
  `問題数 = talkLines.size() / 2`

問題 `i`（0始まり）の対応は次の通り。

- 置換対象テキスト: `talkLines[i]`
- 問題文テキスト: `talkLines[i + 問題数]`

### 3.2 `song` の扱い

- 置換対象は `song.tracks[*].notes[*].lyric` のうち空文字でない要素。
- `position` 昇順で処理する。

---

## 4. 用語

- 置換対象テキスト: `talk` 前半に置く文字列。例: `さかな[が]`
- 問題文テキスト: `talk` 後半に置く表示用文字列。例: `やすいおさかなは？`
- ベース語: 置換対象テキストから `[` と `]` だけを除いた文字列。例: `さかな[が]` -> `さかなが`
- 助詞: `[]` の中身。例: `さかな[が]` -> `が`
- 最大入力音節数: ベース語を音節分割した件数

---

## 5. `[]` 記法

### 5.1 解析仕様（`ParseTargetText`）

- `[` が無い: `baseText = 元文字列`, `particleText = 空`
- `[` と `]` がある:  
  `baseText = 先頭 + []内 + 末尾`  
  `particleText = []内`

例:

- `さかな[が]` -> `baseText = さかなが`, `particleText = が`
- `さかな` -> `baseText = さかな`, `particleText = 空`

### 5.2 使用箇所

- 最大入力音節数計算
- `song` 置換ターゲット
- 入力補完時の助詞追加判定
- Result 表示文字列の助詞付与判定

---

## 6. 問題データ生成（`BuildTalkProblems`）

`talkLines` から `TalkProblem` を作る。

各問題で保持する値:

- `rawTargetText`: `talk` 前半の元文字列
- `baseTargetText`: ベース語
- `particleText`: 助詞
- `questionText`: 問題文
- `targetSyllables`: ベース語の音節配列
- `maxSyllableCount`: `targetSyllables.size()`

---

## 7. 入力処理（`WriteLyrics`）

### 7.1 問題表示

- 問題文は `questionText` を表示する。
- 先頭に `①`, `②`, ... を自動付与する。
- 問題文のフォントサイズは文字数で決める。  
  8文字以下: 100  
  9〜10文字: 90  
  11文字以上: 80

### 7.2 入力制約

- 許可文字: ひらがなと `ー`
- 先頭 `ー` は禁止
- `っ` は禁止
- 音節数制約: `2 <= 入力音節数 <= 問題ごとの最大入力音節数`

### 7.3 長音処理

- `replaceChoonWithVowel` で `ー` を直前母音に変換してから音節処理する。
- 実装上、ここで `ア/イ/ウ/エ/オ` に変換される。

### 7.4 入力補完

入力音節数が最大入力音節数未満のとき、残り枠を補完する。

1. 残り枠 `remaining = max - inputCount` を求める。
2. 助詞があり、助詞音節数 `<= remaining` の場合:
- 先に母音補完を `remaining - 助詞音節数` 回行う。
- 最後に助詞を追加する。
3. 上記以外:
- 母音補完を `remaining` 回行う。

母音補完は「入力末尾音節の母音」に対応するひらがなを使う。

- `a/i/u/e/o` -> `あ/い/う/え/お`
- `N/Q` -> `ん/っ`

### 7.5 タイムアップ

- その問題の最大入力音節数ぶん `ら` を作って回答とする。

### 7.6 `SolvedTask` への保存

- `phrase`: ベース語
- `syllables`: ベース語の音節配列
- `userInput`: ユーザー入力（補完前・表示用）
- `userSyllables`: 置換用音節配列（補完後）

---

## 8. 歌唱用の置換処理（`ApplyParodyLyrics`）

### 8.1 基本

- `task.syllables` と `task.userSyllables` の件数が一致する問題だけ置換する。
- 非一致の問題は置換しない。

### 8.2 重複ターゲット対応

同一ターゲットが複数回出る場合、問題順に曲中出現順へ対応づける。

実装方法:

- 置換前の歌詞列を基準に、ターゲットごとの出現開始位置一覧を先に作る。
- ターゲットごとに「何回目を消費したか」を持つ。
- 問題ごとに、その回数に対応する開始位置へ置換する。

補足:

- 置換後歌詞列で再探索すると、出現番号ずれが起きるため採用しない。

---

## 9. Result 表示用の歌詞生成（`BuildResultDisplayLyrics`）

Result 画面の歌詞は、歌唱用置換結果ではなく、次の表示専用ルールで生成する。

### 9.1 原曲文字列の作り方

- `song.notes` を `position` 昇順で走査する。
- ノート間にギャップがあれば改行（`\n`）を入れる。
- `lyric` は表示補正して連結する。

表示補正:

- `ア/イ/ウ/エ/オ` -> `ー`
- `ァ/ィ/ゥ/ェ/ォ` -> 空文字
- そのほか一部カタカナ音素は表示用ひらがなへ変換（実装の補正表に準拠）

### 9.2 置換の考え方

- 原曲文字列から `phrase`（ベース語）を問題順に削除置換する。
- 置換の出現順は、原曲文字列の出現位置を固定して管理する。

### 9.3 差し込み文字列

基本は `userInput`（補完前入力）。

ただし次をすべて満たすときは `userInput + particleText` を使う。

- その問題に助詞がある
- `userInput` の音節数が問題の最大入力音節数未満
- `userSyllables` の末尾が助詞音節列と一致

意図:

- 母音補完は表示に出さず、助詞だけ表示に反映するため。

---

## 10. 主な関数対応表

- `ExtractTalkUtterances`: `talk` 文字列抽出
- `ParseTargetText`: `[]` 解析
- `BuildTalkProblems`: 問題配列生成
- `WriteLyrics::update`: 入力受付、制約、補完、`SolvedTask` 保存
- `ApplyParodyLyrics`: 歌唱用 `song.notes.lyric` 置換
- `BuildResultDisplayLyrics`: Result 表示用歌詞生成
- `ExtractSongLyrics`: 歌唱確認用の歌詞抽出（休符改行・表示補正あり）

---

## 11. 参照時チェック項目

- `talk` 件数は偶数か
- 問題数 `= talkLines.size()/2` で期待どおりか
- 置換対象テキストと `song` の一致が取れているか
- 同一ターゲット複数回の置換順が崩れていないか
- 助詞付き表示（`入力+助詞`）が意図どおりか
- 休符改行と長音表示（カタカナ母音 -> `ー`）が意図どおりか
